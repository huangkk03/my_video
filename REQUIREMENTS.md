# MKV 视频平台需求规格文档

## 1. 项目概述

**项目名称**: MKV Video Platform  
**项目类型**: 在线视频流媒体平台  
**核心功能**: 支持 MKV 格式视频上传、自动转码为 HLS 流媒体播放、观看进度记忆  
**目标用户**: 视频内容创作者、影视爱好者、在线教育用户

## 2. MVP 功能清单

### 2.1 视频上传模块
- 支持拖拽上传和点击选择文件
- 支持 MKV、MP4、AVI 等常见视频格式
- 上传前进行文件格式校验
- 显示上传进度条
- 支持大文件分片上传（可选）

### 2.2 视频转码模块
- 后端异步转码服务
- 将 MKV 转换为 HLS (.m3u8 + .ts 切片)
- 使用 FFmpeg 进行转码
- 转码进度实时显示
- 支持失败重试机制

### 2.3 视频播放模块
- 支持 HLS 流媒体播放
- 播放器控制条（播放/暂停、音量、进度条、全屏）
- 记住播放位置（进度记忆）
- 支持画质选择（可选）

### 2.4 视频列表模块
- 视频卡片展示（缩略图、标题、时长、状态）
- 视频状态标识（转码中、已完成、失败）
- 分页加载

### 2.5 用户交互
- 进度自动同步（每 5 秒保存一次播放位置）
- 下次打开自动续播

## 3. 非功能需求

### 3.1 性能要求
- 转码服务支持并发处理
- 视频列表支持分页查询
- 播放器首屏加载时间 < 2秒

### 3.2 存储要求
- 原始视频存储路径: `/data/videos/{uuid}/original`
- 转码后 HLS 存储路径: `/data/videos/{uuid}/index.m3u8`

### 3.3 接口规范
- RESTful API 设计
- 支持 HTTP 206 Partial Content（视频流式传输）
- CORS 跨域支持

## 4. 技术栈

- **后端**: Spring Boot 3 + Java 25
- **前端**: Vue 3.5+ + Vite
- **播放器**: ArtPlayer + Hls.js
- **转码工具**: FFmpeg
- **数据库**: MySQL/PostgreSQL

## 5. 页面结构

1. **首页/视频列表页** - 展示所有视频卡片
2. **上传页** - 文件上传入口
3. **播放页** - 视频播放器 + 进度记忆
