改进后的 AList MKV 流式转码优化方案
1. 核心逻辑改进：FFmpeg 命令增强
针对远程 HTTP 流不稳定的情况，在执行命令中加入重连和超时机制，防止因网络瞬时抖动导致 10GB 大文件转码中断。

修改 TranscodeService.java 中的 executeTranscodeFromUrl 方法：

private void executeTranscodeFromUrl(String sourceUrl, Video video, TranscodeTask task) throws Exception {
    Path videoDir = Paths.get(VIDEO_STORAGE_PATH, video.getUuid());
    Files.createDirectories(videoDir);
    Path hlsPath = videoDir.resolve("index.m3u8");
    
    // 改进：增加重连参数，应对 AList 签名抖动或网络波动
    List<String> command = new ArrayList<>(Arrays.asList(
        FFMPEG_COMMAND,
        "-reconnect", "1",               // 开启重连
        "-reconnect_streamed", "1",      // 开启流重连
        "-reconnect_delay_max", "5",     // 最大重连间隔5秒
        "-io_timeout", "10000000",       // IO超时设置（10秒，单位微秒）
        "-i", sourceUrl,                 // 输入 URL
        "-codec:v", "libx264",
        "-preset", "fast",
        "-crf", "23",
        "-codec:a", "aac",
        "-b:a", "128k",
        "-f", "hls",
        "-hls_time", "10",
        "-hls_list_size", "0",
        "-http_seekable", "1",           // 必须开启以支持 MKV 索引读取
        hlsPath.toString()
    ));

    // 进度处理逻辑保持不变...
}

2. 效率优化：一次读取，同时产出
原始方案在转码结束后再次请求 URL 生成缩略图，这会浪费带宽并增加 AList 签名过期的风险。

优化建议： 在 FFmpeg 转码 HLS 的同时直接截取第一帧作为缩略图。

修改后的 FFmpeg 组合命令逻辑：
// 在一次进程调用中完成 HLS 切片和缩略图提取
String[] command = {
    FFMPEG_COMMAND, "-i", sourceUrl,
    // 路径1：转码 HLS
    "-codec:v", "libx264", "-preset", "fast", "-f", "hls", hlsPath.toString(),
    // 路径2：在第5秒处截取一张缩略图
    "-ss", "00:00:05", "-vframes", "1", "-vf", "scale=320:-1", videoDir.resolve("thumbnail.jpg").toString()
};
3. 存储与元数据同步改进
为了支持后续的“任务重试”或“链接刷新”，需要更清晰地标记原始资源状态。

Video 实体增强：

sourceType: 标记为 REMOTE_ALIST。

originalPath: 存储 AList 的相对路径（例如 /Movies/Avatar.mkv），而不是带签名的临时 URL。

原因：带签名的 URL 会过期，存储相对路径可以方便在转码失败时重新生成有效的 raw_url 进行重试。
4. 健壮性保障：异常自动清理
流式转码最怕中途失败留下大量碎片文件。

修改 handleTranscodeFailure 逻辑：
private void handleTranscodeFailure(Video video, TranscodeTask task, Exception e) {
    log.error("Stream transcode failed, cleaning up partial files...");
    try {
        // 清理已生成的 .ts 片段，防止占用磁盘
        Path videoDir = Paths.get(VIDEO_STORAGE_PATH, video.getUuid());
        if (Files.exists(videoDir)) {
            FileUtils.deleteDirectory(videoDir.toFile()); 
        }
    } catch (IOException ioException) {
        log.warn("Failed to cleanup directory: {}", videoDir);
    }
    // 更新数据库状态...
}
