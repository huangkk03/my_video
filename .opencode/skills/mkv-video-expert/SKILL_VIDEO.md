# 在线视频流媒体平台开发需求文档 (PRD)

## 1. 项目概述
构建一个基于 Web 的在线视频点播平台，支持视频上传、管理、在线播放、自动刮削（元数据获取）以及外部网盘挂载。系统需完全容器化部署。

## 2. 技术栈规范
- **前端框架**: Vue 3 + Vite + TypeScript
- **UI 组件库**: Element Plus (后台管理) + Video.js 或 DPlayer (播放器)
- **后端框架**: Spring Boot 3.x
- **数据库**: MySQL 8.0
- **ORM**: MyBatis-Plus
- **视频处理核心**: FFmpeg (用于转码和截图)
- **容器化**: Docker & Docker Compose
- **存储**: 本地磁盘 + 阿里云盘/WebDAV (Rclone集成)

## 3. 系统核心功能模块

### 3.1 视频存储与处理子系统 (核心难点)
- **格式兼容**: 支持上传 `.mp4`, `.mkv`, `.avi`, `.mov` 等常见格式。
- **自动转码 (MKV -> HLS)**:
    - 后端检测到非浏览器原生支持的格式（如 MKV）时，自动调用 FFmpeg 进行转码。
    - **目标格式**: 转码为 HLS (.m3u8) 格式，包含多个清晰度（1080p, 720p, 480p）。
    - **切片存储**: 将视频切片存储在对象存储或本地映射目录中。
- **视频截图**: 自动生成视频封面图（截取第 5 秒画面）。
- **断点续传**: 大文件上传需支持分片上传和断点续传。

### 3.2 视频元数据与刮削子系统
- **在线刮削**:
    - 集成 TMDB (The Movie Database) 或 豆瓣 API。
    - 根据文件名自动搜索并匹配电影/剧集信息（海报、简介、演员、上映年份）。
    - 支持手动修正刮削结果。
- **搜索功能**:
    - 基于 MySQL 全文索引或 Elasticsearch (可选)，支持按标题、演员、标签搜索视频。

### 3.3 网盘挂载与管理
- **网盘集成**:
    - 集成 Rclone 或 AList 核心逻辑。
    - 支持挂载阿里云盘、百度网盘或 WebDAV 协议。
    - **功能**: 后台可直接浏览网盘文件，将网盘中的视频“导入”到本地库（软链接或流式转发），无需完全下载到服务器硬盘。

### 3.4 后台管理系统 (Admin Dashboard)
- **视频库管理**: 列表展示、增删改查、状态监控（转码中/已完成/失败）。
- **分类管理**: 电影、电视剧、动漫、纪录片等分类。
- **用户管理**: 简单的用户权限控制（管理员/普通用户）。

### 3.5 前端播放器 (User Interface)
- **在线播放**: 支持 HLS 流媒体播放，自适应网速。
- **播放记录**: 记录用户播放进度，下次继续观看。
- **响应式设计**: 适配 PC 和 移动端。

## 4. 数据库设计 (MySQL 8.0)

需要设计以下核心表：
1.  **`video_info`**: 存储视频基本信息 (ID, 标题, 描述, 封面URL, 原始文件路径, 播放地址(m3u8), 时长, 大小, 状态)。
2.  **`video_meta`**: 存储刮削信息 (IMDB_ID, TMDB_ID, 导演, 演员JSON, 上映日期)。
3.  **`category`**: 分类表。
4.  **`user`**: 用户表。

## 5. 容器化部署架构 (Docker Compose)

请编写 `docker-compose.yml`，包含以下服务：
1.  **`app-backend`**: Spring Boot 应用。
2.  **`app-frontend`**: Nginx 托管 Vue 静态资源。
3.  **`db-mysql`**: MySQL 8.0 实例。
4.  **`worker-ffmpeg`**: 一个独立的 Worker 容器（或集成在后端中），专门负责监听转码任务队列并执行 FFmpeg 命令。
5.  **`rclone`**: 用于网盘挂载的侧边容器。

## 6. 开发步骤指令 (给 OpenCode 的提示)

1.  **第一步**: 初始化 Spring Boot 项目结构，配置 MySQL 8.0 连接，编写实体类和 Mapper。
2.  **第二步**: 实现文件上传接口，使用 Java `ProcessBuilder` 或 `ffmpeg-cli-wrapper` 封装 FFmpeg 转码逻辑（MKV 转 MP4/HLS）。
3.  **第三步**: 开发视频元数据刮削服务，编写简单的 HTTP 客户端调用公开 API。
4.  **第四步**: 开发 Vue 3 前端，实现视频列表页和后台管理页。
5.  **第五步**: 编写 `docker-compose.yml`，确保所有服务一键启动。

## 7. 注意事项
- **安全性**: 接口需要基本的鉴权（JWT）。
- **性能**: 转码是耗时操作，必须使用异步线程或消息队列（如 Redis Stream 或 简单的数据库状态轮询）处理，不能阻塞主线程。
